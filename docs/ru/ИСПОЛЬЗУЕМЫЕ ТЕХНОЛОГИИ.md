# Используемый стек технологий

Документ перечисляет основные технологии, применяемые в проекте hiTech, и объясняет назначение каждой из них.

## Backend сервисы

### Go Orchestrator
- **Язык:** Go 1.21+
  - Выбор обусловлен производительностью, поддержкой многопоточности и статической типизацией.
- **Фреймворки и библиотеки:**
  - **Gin** (`github.com/gin-gonic/gin`) — лёгкий HTTP-фреймворк для REST API.
  - **gRPC** (`google.golang.org/grpc`) — высокопроизводительный RPC-канал между оркестратором и drone-service.
  - **Prometheus client** (`github.com/prometheus/client_golang/prometheus`) — метрики запросов/задержек.
  - **SQLC + pgx** — типобезопасный доступ к PostgreSQL. SQLC генерирует код репозиториев, `pgx` обеспечивает пул соединений.
  - **RabbitMQ клиент** (`github.com/rabbitmq/amqp091-go`) — публикация задач доставки и подтверждений.
  - **JWT** (`github.com/golang-jwt/jwt`) — токены авторизации.

### Drone Service (Python)
- **Язык:** Python 3.11
- **Фреймворки:**
  - **FastAPI** — REST API для health-check, метрик и административных операций.
  - **Pydantic** — валидация моделей, управление конфигурацией.
  - **gRPC** (`grpcio`) — взаимодействие с оркестратором по операциями с ячейками/статусами.
  - **WebSockets** (`websockets` / FastAPI) — реальное время для управления дроном и трансляции статусов.
  - **RabbitMQ** (`aio-pika`/кастомные обёртки) — очередь задач (`deliveries`, `delivery.return`).
  - **Prometheus client** (`prometheus_client`) — метрики FastAPI.
  - **Asyncio** — обработка фонов.

### Parsers (аппаратные интеграции)
- **Drone Parser:**
  - **ROS (Robot Operating System)** — телеметрия, сервисы навигации, детекция ArUco.
  - **Clover API** — простые команды для взлёта/посадки.
  - **Собственный контроллер навигации** — логика доставки, работа с ArUco-картой.
- **Parcel Automat Parser:**
  - Python-сервис для управления ячейками, QR-сканером и связью с оркестратором.
  - **Arduino скетч** (`arduino_controller.ino`) — управление замками/датчиками.

## Frontend

### Админ-панель
- **React 18 + TypeScript** — SPA для операторов.
- **Vite** — девсервер и сборка.
- **Font Awesome** — иконки.
- **Context API** — управление авторизацией/состоянием.
- **Docker** — контейнеризация.

### Мобильное приложение (Flutter)
- **Flutter + Dart** — кроссплатформенный клиент (Android/iOS/Web).
- **Clean Architecture** — разделение на слои `data`, `domain`, `presentation`.
- **HTTP клиент** — обёртка с перехватчиками для авторизации.
- **Push** — интеграция с Firebase FCM (опционально).

## Данные и очередь
- **PostgreSQL** — основная БД (заказы, доставки, пользователи, товары).
- **MinIO** — S3-совместимое хранилище (QR, медиа).
- **RabbitMQ** — брокер сообщений для задач доставки, отмен, подтверждений.

## Мониторинг
- **Prometheus** — сбор метрик.
- **Grafana** — дашборды и визуализация.
- **Loki + Promtail** — централизованное логирование.
- **Alertmanager (опционально)** — цепочка уведомлений.
- **cAdvisor / node-exporter** — метрики контейнеров и хоста.
- **RabbitMQ / Postgres exporters** — дополнительные метрики.

## Инфраструктура
- **Docker Compose** — запуск мультисервисных окружений.
- **Nginx** — reverse proxy, TLS.
- **Makefile** — команды сборки/запуска для Go.
- **Virtualenv / requirements.txt** — повторяемость Python окружения.
- **Flutter CLI** — сборка мобильных клиентов.

## Безопасность
- **JWT** — авторизация пользователей.
- **SMSAero API** — отправка кодов подтверждения.
- **MinIO access keys** — защищённый доступ к объектам.
- **TLS (Nginx)** — рекомендовано для прод.

## Тестирование и качество
- **Go:** `go test ./...`.
- **Python:** `pytest`.
- **Flutter:** `flutter test`.
- **Статический анализ:** Go Vet, Staticcheck, mypy, ruff, dart analyze (по необходимости).

## Итог
Стек сочетает высокопроизводительный Go-оркестратор, Python-логику для аппаратуры и современные клиенты (web + mobile) с полной инфраструктурой мониторинга и деплоя на Docker Compose.
