# Структура монорепозитория hiTech

> Репозиторий оформлен как монорепо: включает серверные сервисы, парсеры для оборудования, административную веб-панель, мобильное приложение, конфигурации деплоя и документацию. Этот документ подробно описывает назначение основных директорий и ключевых модулей.

## Общий обзор

| Путь | Назначение |
| --- | --- |
| `admin_panel/` | Административная панель (React + Vite) для операторов: мониторинг доставок, управление товарами, постаматами, дронами и пользователями. |
| `backend/` | Серверная часть: Go оркестратор, Python drone-service и парсеры для дронов/постаматов. |
| `deployment/` | Docker Compose конфигурации и Nginx-шаблоны для dev/prod окружений. |
| `docs/` | Документация на EN/RU, архитектурные схемы. |
| `mobileapp/` | Мобильное приложение на Flutter для курьеров/клиентов. |
| `monitoring/` | Конфигурации Prometheus, Grafana, Loki, Promtail. |
| `proto/` | Общие gRPC-протофайлы. |
| `env.example` | Шаблон для `.env` / `.env.dev`. |

## Backend (`backend/`)

### `go-orchestrator/`
- **Язык:** Go (Gin, gRPC).
- **Роль:** Центральный API: авторизация, заказы, товары, доставки, управление ячейками, интеграция с drone-service и постаматами.
- **Важно:**
  - `cmd/app/main.go` — точка входа HTTP + gRPC.
  - `internal/controller/http` — Gin-хендлеры (версия `v1`, разделы по доменам).
  - `internal/controller/grpc` — gRPC-сервер (открытие ячеек, статусы).
  - `internal/usecase` — бизнес-логика, публикация задач, проверка статусов.
  - `internal/usecase/repo` — репозитории на базе `sqlc` (PostgreSQL).
  - `pkg/pb` — сгенерированные gRPC клиенты.
  - `migrations/` — миграции БД.
  - `config/` — загрузка конфигурации из env/yaml.

### `drone-service/`
- **Язык:** Python (FastAPI, gRPC, WebSockets).
- **Роль:** Промежуточный слой между оркестратором и физическими дронами. Управляет задачами доставки, хранит состояние дронов, обрабатывает WebSocket-сессии.
- **Ключевые директории:**
  - `app/api` — FastAPI роуты, схемы, метрики.
  - `app/presentation` — WebSocket обработчики (дроны, админ панель).
  - `app/core` — use case'ы, менеджер дронов, сущности, state repository.
  - `app/infrastructure` — адаптеры (RabbitMQ клиент, Postgres-хранилище состояния).
  - `app/services/workers` — фоновые потребители RabbitMQ (доставки, возвраты).
  - `config/` — pydantic-настройки.

### `parsers/`
Аппаратура и низкоуровневые скрипты.
- `drone/`
  - `app/navigation` — Clover API, ArUco навигация.
  - `app/services` — WebSocket клиент, логика доставки.
  - `scripts/` — standalone-скрипты ROS (`takeoff.py`, `land.py`, `navigate_to_aruco.py`) для `rosrun`.
  - `test_scripts/` — диагностические полёты (`simple_flight_test.py`).
  - `config/` — карта ArUco, конфиг дрона.
- `parcel_automat/`
  - Python-контроллер постамата: работа с QR, открытие ячеек, связь с оркестратором.
  - `arduino_controller.ino` — прошивка для MCU.

## Admin Panel (`admin_panel/`)
- **Стек:** React, TypeScript, Vite.
- **Структура:**
  - `src/pages/` — страницы (Dashboard, Drones, Goods, Monitoring и др.).
  - `src/components/` — переиспользуемые компоненты (Button, Modal, Layout).
  - `src/api/` — обёртки над REST API оркестратора.
  - `src/context/AuthContext.tsx` — состояние авторизации.
  - `src/config/api_config.ts` — URLы (ENV `VITE_*`).
  - `dist/` — production сборка.

## Mobile App (`mobileapp/`)
- **Фреймворк:** Flutter.
- **Организация:**
  - `lib/core/` — DI, HTTP клиент, темы, push-уведомления.
  - `lib/features/` — фичи в стиле Clean Architecture (`auth`, `goods`, `orders`, `qr`, `notifications` и т.д.).
  - `assets/`, `android/`, `ios/`, `web/` — ресурсы и платформенные проекты.
  - `test/` — тесты.

## Документация (`docs/`)
- Директории `en/`, `ru/` — зеркальные файлы (`API-SPEC.md`, `DEPLOYMENT.md`, `OBSERVABILITY.md`, `STRUCTURE.md`). Английская версия создаётся первой, затем перевод.
- `arch.png` — архитектурная схема.

## Deployment (`deployment/`)
- `docker/`
  - `docker-compose.dev.yml` — локальный стек с мониторингом.
  - `docker-compose.prod.yml` — прод-конфигурация за reverse proxy.
  - `docker-compose.npm.yml` — вариант под Nginx Proxy Manager.
- `nginx/`
  - Конфиги маршрутизации (dev/prod) для API, админки, drone-service, MinIO, Grafana.

## Monitoring (`monitoring/`)
- `prometheus/` — конфиг Prometheus (`prometheus.yml`), файлы алертов.
- `grafana/` — дашборды и provisioning.
- `loki/`, `promtail/` — логирование.
- Подробности: `docs/ru/OBSERVABILITY.md`.

## Общие ресурсы
- `proto/orchestrator.proto` — gRPC протокол между оркестратором и drone-service. Генерируется в Go (`pkg/pb`) и Python (в `backend/drone-service/app/proto`).
- `env.example` — шаблон переменных окружения (JWT секреты, Postgres, RabbitMQ, Grafana, MinIO и т.д.).

## Советы для разработки
- **Go:** `make run`, `go test ./...`. Репозитории генерируются через `sqlc`.
- **Python:** Uvicorn (`python main.py`), тестирование `pytest`. Работает в virtualenv/Poetry.
- **ROS-скрипты:** перед запуском выполните `source` рабочего пространства ROS; далее `rosrun clover <скрипт>.py`.
- **Flutter:** `flutter run`, `flutter test`.
- **Docker:** `docker compose -f deployment/docker/docker-compose.dev.yml up` — полный стек.
