# Руководство по мониторингу и логированию hiTech

## Обзор
- **Логирование:** Сервисы пишут структурированные логи в stdout. Promtail собирает их и отправляет в Loki для централизованного поиска.
- **Метрики:** Каждый сервис отдаёт метрики в формате Prometheus. Prometheus опрашивает цели, Grafana визуализирует дашборды.
- **Алерты:** Prometheus поставляется с заготовкой правил (`monitoring/prometheus/alerts.yml`). Дополняйте и подключайте Alertmanager, когда потребуется продакшн-уведомления.

## Конвейер логов

### Схема работы
1. Сервисы (Go orchestrator, Python drone-service, admin panel, Postgres, MinIO) пишут логи в stdout/stderr.
2. `promtail` (см. `monitoring/promtail/promtail-config.yml`) использует Docker socket discovery и читает контейнеры с ярлыком `com.docker.compose.service`, указанным в конфиге.
3. Promtail добавляет ярлыки к каждому сообщению:
   - `container_name`
   - `service` (имя сервиса в Compose)
   - общие метки (`environment`, `compose_project`), которые Docker Compose выставляет автоматически.
4. Логи отправляются в Loki по адресу `http://loki:3100` (внутренний контур).
5. Grafana читает Loki через раздел Explore или дашборды.

### Покрытие сервисов
| Сервис | job в Promtail | Примечания |
| --- | --- | --- |
| Go orchestrator | `go-orchestrator` | Использует пакет `log`. Уровень регулируется `LOG_LEVEL` (см. `.env`). |
| Drone service | `drone-service` | Python-модуль `logging`, по умолчанию INFO. |
| Admin panel | `admin-panel` | Полезно для диагностики сборки фронтенда. |
| Postgres | `postgres` | Можно управлять объёмом через `POSTGRES_LOG_STATEMENT`. |
| MinIO | `minio` | Журналы доступа к хранилищу.

Чтобы добавить новый контейнер, продублируйте блок в `promtail-config.yml` и измените фильтр `com.docker.compose.service`.

### Хранилище Loki и ретенция
- Хранилище: локальная файловая система `/loki/{chunks,rules}` (`monitoring/loki/loki-config.yml`).
- Ретенция: по умолчанию `retention_enabled: false`. Перед продакшном включите `compactor.retention_enabled` и задайте сроки в `limits_config`.
- Доступ: Grafana (порт `GRAFANA_PORT`, стандарт `3000`) → вкладка *Explore* → источник данных `Loki`.

### Быстрые запросы
| Задача | Пример LogQL |
| --- | --- |
| Ошибки оркестратора | `{service="go-orchestrator"} |= "ERROR"` |
| События по WebSocket дрона | `{service="drone-service"} |= "websocket"` |
| Логи сборки админки | `{service="admin-panel"}` |

## Конвейер метрик

### Цели и порты
Конфигурация Prometheus находится в `monitoring/prometheus/prometheus.yml`. Базовый интервал опроса — 15 секунд.

| Job | Target | Порт | Назначение |
| --- | --- | --- | --- |
| prometheus | `localhost:9090` | 9090 | Самомониторинг |
| go-orchestrator | `go-orchestrator:9091` | 9091 | HTTP и gRPC счётчики/гистограммы |
| drone-service | `drone-service:9092` | 9092 | Метрики FastAPI и WebSocket |
| rabbitmq | `rabbitmq:15692` | 15692 | Требует включённого плагина Prometheus |
| cadvisor | `cadvisor:8080` | 8080 | Нагрузка контейнеров |
| node-exporter | `node-exporter:9100` | 9100 | Метрики хоста |
| postgres | `postgres-exporter:9187` | 9187 | Нужны переменные `POSTGRES_EXPORTER_*` |

### Инструментация приложений
- **Go orchestrator:** `internal/controller/http/middleware/prometheus.go` и `internal/controller/grpc/middleware/prometheus.go` отдают счётчики (`hitech_http_requests_total`) и гистограммы задержек (`hitech_http_request_duration_seconds`) на `/metrics` (порт 9091).
- **Drone service:** `app/api/prometheus.py` регистрирует `fastapi_requests_total` и `fastapi_request_duration_seconds`. Эндпоинт `/metrics` на порту 9092.
- **Пользовательские метрики:** добавляйте `prometheus` библиотеки (`github.com/prometheus/client_golang/prometheus` или `prometheus_client`) и регистрируйте счётчики/гистограммы — Prometheus подберёт их автоматически.

### Дашборды Grafana
- Файлы: `monitoring/grafana/dashboards/` (JSON-экспорты). Импортировать через Grafana → `Dashboards → Import`.
- Учётные данные: `GRAFANA_USER` / `GRAFANA_PASSWORD` (см. `env.example`, строки 60-61). Для продакшна сменить сразу.
- Источники данных:
  - `Prometheus` — URL `http://prometheus:9090`.
  - `Loki` — URL `http://loki:3100`.

### Алерты и уведомления
- Правила: `monitoring/prometheus/alerts.yml` — стартовые шаблоны. Добавьте собственные пороги (ошибки, задержки, очередь задач).
- Подключите Alertmanager: добавьте блок `alertmanager` в `prometheus.yml`, настройте уведомления в Grafana (email, Slack и т.п.).

## Операционные советы
- **Контроль объёма логов:** регулируйте уровни через переменные (`LOG_LEVEL`, `DRONE_LOG_LEVEL`). В Go можно оборачивать `log.Println` проверками.
- **Диск:** Loki хранит чанки локально — следите за объёмом `/loki` (cAdvisor или node-exporter).
- **Резервные копии:** Метрики обычно не бэкапятся. При необходимости снимайте снапшоты каталога `/prometheus`.
- **Безопасность:** По умолчанию Prometheus и Loki доступны только внутри сети Compose. Для внешнего доступа используйте reverse proxy и авторизацию.
- **URL для dev-стека:**
  - Prometheus: `http://localhost:${PROMETHEUS_PORT:-9090}`
  - Grafana: `http://localhost:${GRAFANA_PORT:-3000}`
  - Loki API: `http://localhost:${LOKI_PORT:-3100}`

## Расширение стека
1. **Новая цель метрик:**
   - Экспортируйте `/metrics` в сервисе.
   - Добавьте `job` в `monitoring/prometheus/prometheus.yml`.
   - Перезапустите Prometheus: `docker compose restart prometheus`.
2. **Новый источник логов:**
   - Убедитесь, что контейнер имеет ярлык `com.docker.compose.service=<name>`.
   - Добавьте job в `promtail-config.yml`.
   - Перезапустите Promtail: `docker compose restart promtail`.
3. **Дашборды как код:** экспортируйте JSON и коммитьте в `monitoring/grafana/dashboards`.
